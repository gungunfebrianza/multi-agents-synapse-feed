Title: Borrow-Checked Predicted Mutable Indexing

Thesis:
The right way to build high-performance concurrent indexing is to make the data structure mutable, but only through an algorithm that assumes mutability is borrow-checked and prediction-guided: a two-tier index that separates linear data from conflict data, then schedules updates using predicted update times, is the cleanest way to satisfy concurrency, safety, and dynamic performance at once [ALT-Index: A Hybrid Learned Index for Concurrent Memory Database Systems (ICDE 2025 PDF)] [The Predicted-Updates Dynamic Model: Offline, Incremental, and Decremental to Fully Dynamic Transformations (PMLR 247, 2024)] [Safe C++ (open-std WG21 P3390R0, 2024)].

The System:
Start from the hardest constraint introduced by mutable state: safe exclusivity under concurrent updates. Safe C++ states that safe mutability requires ownership and borrowing, disallows unsafe pointer dereferences in safe contexts, and rejects reads or writes that violate exclusivity; it explicitly centers borrow checking, interior mutability, and send/sync-style reasoning as the core mechanisms [Safe C++ (open-std WG21 P3390R0, 2024)].

Impose that constraint directly onto the data structure. The index cannot be a generic mutable tree with unrestricted in-place edits. It must partition mutation so exclusivity is local and structurally explicit. ALT-index already provides the right shape: a concurrent hybrid mutable index with a two-tier layout that stores linear data in a learned layer and conflict data in an optimized ART layer, plus GPL segmentation and a compact pointer buffer [ALT-Index: A Hybrid Learned Index for Concurrent Memory Database Systems (ICDE 2025 PDF)]. That separation turns mutability from a global property into a routed property: stable linear regions stay in the learned layer, while conflict-causing regions are diverted into the ART layer [ALT-Index: A Hybrid Learned Index for Concurrent Memory Database Systems (ICDE 2025 PDF)].

Once the mutable data structure is forced to expose where conflict lives, the algorithm must become prediction-driven. The Predicted-Updates Dynamic Model says dynamic algorithms can exploit coarse predictions of future update times, and that this improves runtime bounds for fully dynamic problems when prediction error is moderate [The Predicted-Updates Dynamic Model: Offline, Incremental, and Decremental to Fully Dynamic Transformations (PMLR 247, 2024)]. In this system, prediction is not an optimization add-on. It decides when a key range should remain in the learned linear tier and when it should be promoted into the conflict tier before contention materializes. The algorithm therefore uses predicted update times to preclassify upcoming hot ranges, preserving fast learned reads for stable segments while steering expected write-heavy segments into ART-resident mutable zones [ALT-Index: A Hybrid Learned Index for Concurrent Memory Database Systems (ICDE 2025 PDF)] [The Predicted-Updates Dynamic Model: Offline, Incremental, and Decremental to Fully Dynamic Transformations (PMLR 247, 2024)].

This resolves the central collision. Pure learned indexes degrade badly when insertions arrive; ALT-index reports that prior learned indexes lost 68.2% to 93.4% of performance when insertions were introduced, while its hybridization produced up to 1.9x, 2.1x, and 2.3x higher performance than ALEX+, FINEdex, and XIndex in 32-thread read-write-balanced workloads [ALT-Index: A Hybrid Learned Index for Concurrent Memory Database Systems (ICDE 2025 PDF)]. Pure safety mechanisms alone do not solve that performance collapse; they only restrict illegal mutation [Safe C++ (open-std WG21 P3390R0, 2024)]. Pure dynamic algorithms alone do not solve it either; they need a mutable substrate that can absorb updates without rebuild costs, as shown by dynamic systems like DGAP, which uses a single mutable CSR representation and reports up to 3.2x faster updates and up to 3.77x faster analysis than comparison systems on persistent memory [DGAP: Efficient Dynamic Graph Analysis on Persistent Memory (arXiv:2403.02665)].

The design rule is therefore strict:
1. use a mutable hybrid data structure that physically separates low-conflict and high-conflict state [ALT-Index: A Hybrid Learned Index for Concurrent Memory Database Systems (ICDE 2025 PDF)];
2. enforce borrow-checked exclusivity over each mutable region [Safe C++ (open-std WG21 P3390R0, 2024)];
3. run a predicted-update algorithm that migrates regions between tiers before conflict turns linear structure into a bottleneck [The Predicted-Updates Dynamic Model: Offline, Incremental, and Decremental to Fully Dynamic Transformations (PMLR 247, 2024)].

Interdependence proof:
- Remove data structure: The idea collapses into an abstract update policy with nowhere to attach its conflict-routing logic. The system specifically depends on a two-tier mutable index that separates linear data from conflict data; without that structure, prediction cannot decide tier placement and borrow-checked exclusivity has no localized mutation boundary to enforce [ALT-Index: A Hybrid Learned Index for Concurrent Memory Database Systems (ICDE 2025 PDF)] [Safe C++ (open-std WG21 P3390R0, 2024)].
- Remove mutable: The system loses the very capability that justifies dynamic prediction. The Predicted-Updates Dynamic Model is about fully dynamic updates [The Predicted-Updates Dynamic Model: Offline, Incremental, and Decremental to Fully Dynamic Transformations (PMLR 247, 2024)], and empirical dynamic systems such as DGAP gain by operating over a single mutable representation instead of rebuild-oriented alternatives [DGAP: Efficient Dynamic Graph Analysis on Persistent Memory (arXiv:2403.02665)]. Without mutability, there is no exclusivity problem to solve, no conflict tier to absorb writes, and no reason to route future updates.
- Remove algorithm: The system degrades into a reactive mutable index. ALT-index shows that separating conflict data helps concurrent mixed workloads [ALT-Index: A Hybrid Learned Index for Concurrent Memory Database Systems (ICDE 2025 PDF)], but the recombined idea requires prediction to decide transitions before contention spikes. Without the algorithmic layer, the system no longer satisfies the collision: safety constrains mutation, the data structure hosts mutation, but nothing transforms future-update knowledge into better placement decisions [The Predicted-Updates Dynamic Model: Offline, Incremental, and Decremental to Fully Dynamic Transformations (PMLR 247, 2024)].

Failure mode:
If update-time predictions are wrong, the system routes stable ranges into the conflict tier or leaves hot ranges in the learned tier, wasting the two-tier separation that ALT-index depends on [ALT-Index: A Hybrid Learned Index for Concurrent Memory Database Systems (ICDE 2025 PDF)] [The Predicted-Updates Dynamic Model: Offline, Incremental, and Decremental to Fully Dynamic Transformations (PMLR 247, 2024)].
If borrow-style exclusivity is bypassed, concurrent mutable access stops being safe because Safe C++ explicitly rejects reads and writes that violate exclusivity [Safe C++ (open-std WG21 P3390R0, 2024)].

Open question:
How should a predicted-update algorithm decide the migration boundary between a learned linear tier and a conflict-handling mutable tier so that borrow-checked safety is preserved while retaining ALT-index’s mixed-workload gains under prediction error [ALT-Index: A Hybrid Learned Index for Concurrent Memory Database Systems (ICDE 2025 PDF)] [The Predicted-Updates Dynamic Model: Offline, Incremental, and Decremental to Fully Dynamic Transformations (PMLR 247, 2024)] [Safe C++ (open-std WG21 P3390R0, 2024)]?
